2. 代替一次資料を検討：
   - 国東市文化財保護条例（g-reiki.net）→ 別表なし、条例本文のみで個別指定物件の記載なし（WebFetchで確認、市サイトへのリクエストは消費せず）。
   - 文化財保存活用地域計画PDF・data.bodik.jpオープンデータ・大分県文化財DB → 該当なし（WebSearchで確認、市指定の個別リストは見つからず）。

   - `bunpu.html`（分布状況）→ 404。
   → 代替なし。大字別ページ巡回が唯一の経路と判断。
3. `siteibunkazai.html`（文化財トップ）から大字別ページ69件超のインデックス（実際には71ページ、地区名の重複・分割ページ含む）を取得し、全件を`WebFetch`で系統的に巡回。curlでの直接アクセスはIncapsula WAFに阻まれ404偽装ページが返るため使用せず、WebFetch経由で全件成功。
4. 各ページから「指定番号・名称・種別・指定年月日（和暦）・員数・所在地・所有者」を一字一句そのまま抽出するようプロンプトで指示し、国・県指定行は除外指示。
5. 和暦→西暦ISO変換はスクリプトで機械的に実施（検算済み、null無し）。

## リクエスト消費
- 対象サイトへの直接アクセス: hub 2回（curl、404確認用） + WebFetch 3回（hub関連ページ確認）+ WebFetch 71回（大字別ページ全件）= 約76リクエスト。予算140に対し十分な余裕内。
- 同一ホストへの直接curlは計3回のみ（内2回は404/robots確認）。WebFetchは自動的にペース制御されるツール経由のため、追加のsleep制御は行っていない。

## 未解決事項（正直な申告）
- 史跡カテゴリが公式39件に対し抽出45件で+6、天然記念物が公式10件に対し11件で+1、有形文化財が公式250件に対し248件で-2。ネット+5件が総数の差分322-317と一致。
- 原因仮説（未検証）:
  1. 各大字ページの「更新日」が2023年6月〜のものが多く、市トップページの集計表（令和7年4月1日更新）より古い可能性がある＝ページ間で更新タイミングのズレがあり、新規指定・登録抹消が未反映の可能性。
  2. 71ページの抽出は各ページ単位でLLM（WebFetch内の要約モデル）に一字一句抽出を指示したが、機械的な誤分類（美術工芸/建造物を史跡と誤読、など）が数件発生している可能性を否定できない。
  3. 大字名が分割されている地区（例: 泉福寺関連＝横手（泉福寺）、行入寺関連＝横手（行入寺）、田深/田深2など）で、同一物件が別ページに重複記載されている可能性は個別に確認したが（鶴川ページの「石器類」「ワラミノ遺跡出土品」に安国寺/大恩寺への移設注記あり）、該当2件は美術工芸品に分類しており史跡の重複要因ではない。
  4. 全322件をHTML原文と1件ずつ突合する検証は未実施（71ページ×平均4.5件の突合が必要でリクエスト予算・時間の制約内では今回省略）。
- 上記の理由により、322件の抽出データはそのまま`out/大分県/kunisaki.jsonl`に格納したが、**史跡39件・天然10件・有形250件への機械的な絞り込みは行っていない**（null-over-guessの原則により、根拠のない削除はしていない）。件数差の解消には、大字別ページの生HTML再取得による全件突合、または市教育委員会への直接照会が必要。
- 除外対象（国・県指定、国選択無民、県選択無民）として各ページで報告された項目は本jsonlに含めていない（除外件数は各ページごとに記録済みだが本ファイルには集計していない）。
